# HIS团队协作调度

> ⚠️ **前置条件**：首次使用前需确保 `wecom_mcp` 工具已加入白名单。执行 `wecom-preflight` 技能检查。

## 功能说明

当PM收到以下关键词时，自动发送任务通知到群里：

| 关键词 | 通知内容 |
|-------|---------|
| 全部/所有人 | 📢 召集全体成员响应 |
| 架构 | 请架构师处理架构相关任务 |
| 后端 | 请后端开发处理后端相关任务 |
| 前端 | 请前端开发处理前端相关任务 |
| 数据库 | 请数据库专家处理数据库相关任务 |
| 测试 | 请测试专家处理测试相关任务 |
| 医疗 | 请医疗专家处理医疗业务相关任务 |
| 文档 | 请文档专家处理文档相关任务 |

## 使用方法

### 步骤1：识别关键词

当用户发送包含关键词的消息时，PM识别并提取：
- 关键词类型
- 具体任务描述（如果有）
- 当前群聊ID

### 步骤2：构建通知消息

根据关键词构建通知内容：

**示例 - "全部"关键词**：
```
📢 【团队召集通知】

任务：{用户提供的任务描述}

请以下成员响应：
@HIS系统架构师 @HIS后端开发 @HIS前端开发 @HIS数据库专家 @HIS测试专家 @HIS医疗专家 @HIS文档处理专家

---
💡 提示：请手动@对应机器人以触发响应
```

**示例 - "架构"关键词**：
```
📋 【架构任务通知】

任务：{用户提供的任务描述}

请 @HIS系统架构师 响应

---
💡 提示：请手动@对应机器人以触发响应
```

### 步骤3：发送消息

使用 `wecom_mcp` tool 发送消息到群里：

```
wecom_mcp call msg send_message '{"chat_type": 2, "chatid": "{群聊ID}", "msgtype": "text", "text": {"content": "{构建的通知内容}"}}'
```

## 重要限制

⚠️ **企微API限制**：机器人发送的消息中@其他机器人，**不会触发被@机器人的回调**。这是企微的硬性限制，无法绑过。

**解决方案**：
- PM发送通知消息并列出@列表
- 用户看到通知后，手动@对应的机器人
- 被@的机器人收到回调并响应

## 工作流程示例

用户：`@HIS项目经理 后端 数据库连接池配置有问题`

PM执行：
1. 识别关键词："后端"
2. 提取任务："数据库连接池配置有问题"
3. 构建通知：
   ```
   📋 【后端任务通知】

   任务：数据库连接池配置有问题

   请 @HIS后端开发 响应

   ---
   💡 提示：请手动@HIS后端开发以触发响应
   ```
4. 发送消息到群里

用户看到通知后：手动`@HIS后端开发 数据库连接池配置有问题`

后端机器人收到回调并响应。
